# ‚è∞ Cron Setup Script ‚Äì Auto Scheduler

This script installs a cron job to automatically run the main scraper pipeline (`runner.py`).

---

## üîπ What it does
1. Detects the absolute path of `runner.py`.  
2. Creates a cron entry that runs **every 10 days at 06:00**.  
3. Checks if a cron job for `runner.py` already exists ‚Äì avoids duplicates.  
4. Installs the cron job in the current user‚Äôs crontab.  

---

## üîπ Usage
Run once to install the cron job:
```bash
python3 utils/setup_cron.py

## üî∏ Final Note ‚Äì Manual Scheduling (optional)

If you prefer to **set the schedule manually** instead of using `utils/setup_cron.py`, you can run **one** of the following commands **from anywhere** in your terminal.  
Each command **overwrites the entire crontab** with a single entry that runs `runner.py` at the specified interval.

> Replace `/path/to/project` with your actual project path.

### Run DAILY at 00:00 (every day)
echo "0 0 * * * /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -

### Run EVERY 2 DAYS at 00:00
echo "0 0 */2 * * /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -

### Run EVERY 7 DAYS at 00:00 (best practice: once a week, every Monday)
echo "0 0 * * 1 /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -
# If you really want "*/7" on the day-of-month field:
# echo "0 0 */7 * * /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -
# ‚ö† Not exactly every 7 days, varies with month length.

### Run EVERY 30 DAYS at 00:00 (best practice: first day of each month)
echo "0 0 1 * * /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -
# If you insist on "*/30" in the day-of-month field:
# echo "0 0 */30 * * /usr/bin/python3 /path/to/project/runner.py >> /path/to/project/cron.log 2>&1" | crontab -
# ‚ö† May be irregular because months have 28‚Äì31 days.

**Behavior with manual config:**  
- Once you install a **manual schedule**, the program will **no longer rely on the default ‚Äúevery 10 days at 06:00‚Äù** set by `utils/setup_cron.py`.  
- The setup script checks if a cron job for `runner.py` already exists and **will not add the 10-day schedule** if a custom one is present.

**Useful commands**
- Show current cron jobs: `crontab -l`  
- Edit manually: `crontab -e`  
- Remove all cron jobs (careful): `crontab -r`
